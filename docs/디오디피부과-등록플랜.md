## 2. 지역(로컬 SEO) 전략 — "강남 청담 피부과"

디오디는 **서울 강남구 청담동**에 위치하므로 메디록 지역 라우트는:

```
/hospitals/서울/강남구/피부과     ← 이 리스트 페이지에 노출
/hospital/디오디피부과의원청담      ← 상세 프로필 (신규)
```

**타깃 키워드**: 청담 피부과 · 강남 피부과 · 압구정 안티에이징 · 청담 줄기세포 · 강남 리프팅(울쎄라/써마지) · 청담 쥬베룩

**등록 전 보장해야 할 region 레코드** (예온의 `당하동` 추가와 동일 패턴):

| slug | nameKr | level | parentSlug |
|---|---|---|---|
| 서울 | 서울 | sido | (빈값) |
| 강남구 | 강남구 | sigungu | 서울 |
| **청담동** | 청담동 | dong | 강남구 |

> `서울`·`강남구`는 홈에서 이미 `/hospitals/서울/강남구/치과`로 링크되어 존재할 가능성이 높습니다. **`청담동`(dong)은 신규 추가** 필요(동 필터·주소 정합용). 진료과 `피부과`(dermatology)는 기존 스키마에 존재.

### 가까운 지하철역 (확인 완료)

좌표 검증: 구글지도 기준 **37.5239416, 127.0421664** (도산대로 423, 청담 명품거리 구간).

| 순위 | 역 | 출구·도보 | medirok 노선(stations.ts) |
|---|---|---|---|
| **1 (최단)** | **압구정로데오역** (수인분당선) | **4번 출구 · 도보 8분** | `분당선` |
| 2 | 강남구청역 (7호선) | 4번 출구 · 도보 15분 | `7호선` / `분당선` |

→ 등록값: `nearestStationName: "압구정로데오역"`, `walkingMinutes: 8`.

**역 기반 SEO 라우트도 자동 연결됩니다.** medirok은 `nearestStationName`이 `stations.ts`의 역명과 일치하면 역주변 페이지·sitemap에 노출됩니다(`getHospitalsByDeptAndRegion`과 별개). 압구정로데오역은 medirok에 **`분당선`** 노선으로 등록돼 있어:

```
/hospitals/역/분당선/압구정로데오역   ← 디오디가 이 역주변 페이지에도 노출
```

즉 "청담 피부과"(지역) + "압구정로데오역 피부과"(역세권) 두 경로로 동시 타깃 가능. 압구정로데오 = 청담 명품거리 상권이라 안티에이징/피부과 검색의도와 정합성이 높습니다.

**※ 이번에 작업한 지역 리스트 페이지 SEO 개선(P0~P2)과 직접 맞물립니다.** 디오디가 `/hospitals/서울/강남구/피부과`에 등록되면, 그 페이지의 ItemList JSON-LD·"인천 서구 치과"식 시·도 포함 메타·동적 FAQ 효과를 그대로 받습니다.

## 4. 메디록 4단계 인증 & 티어 판단

예온과 동일하게 **검증 가능한 사실만** 기입(미검증 단계는 false).

| 단계 | 디오디 상태 | 기입안 |
|---|---|---|
| stage1 진료이력 | ✅ **국내 피부과 최초 보건복지부 지정 첨단재생의료실시기관** · 청담 안티에이징 전문 | true |
| stage2 실방문후기 | ⚠️ 미수집 | false / "실방문 후기 수집 진행 중" |
| stage3 의료진자격 | ✅ 피부과 전문의(이준 외) | true (의료진 확정 후) |
| stage4 시설장비 | ✅ 줄기세포센터 · 메타뷰 3D진단 · 울쎄라/써마지 등 | true |

**티어**: 디오디는 큐레이션 후보(하이엔드)지만, 메디록 비즈니스 모델상 **PREMIUM = 유료 큐레이션 파트너**입니다. → **PREMIUM 등재는 영업/계약 확정 후** 결정. 계약 전이라면 우선 `STANDARD`(디렉터리)로 등재하고, 파트너 전환 시 `tier`+`curationNote` 추가를 권장합니다.

## 5. 등록 방식 (2가지)

**A. 시드 스크립트 (권장 — 멱등·버전관리)**
`scripts/seed-dod.ts`를 `seed-yeon.ts` 복제로 작성 → region(청담동) + hospital upsert.
`package.json`에 `"seed:dod": "node --env-file=.env.local --import tsx/esm scripts/seed-dod.ts"` 추가 후 실행.
장점: 재실행 안전, 데이터가 코드로 남아 리뷰·복구 용이.

**B. Payload 어드민 직접 입력** (`/admin` → Hospitals → Create)
장점: 코드 없이 가능. 단점: 의료진/후기 배열 입력이 번거롭고 이력 추적 어려움.
