# 매거진 본문 마크다운 렌더링 개선 플랜 (옵션 A)

작성일: 2026-06-21
대상: `src/app/(frontend)/magazine/[slug]/page.tsx` 본문(`magazine.body`) 렌더링
증상: 라이브에서 표(`| ... |`)·인라인 `**굵게**`·링크가 마크다운 원문 그대로 노출
(예: https://www.medirok.com/magazine/guide-geomdan-dental-2026)

---

## 1. 원인

본문은 정식 마크다운 파서가 아니라 페이지 내 **수작업 라인 파서**로 렌더됩니다
(`magazine.body.split("\n").map(...)`, 현재 170~205행 부근). 처리 항목이 제한적:

| 입력 | 현재 처리 | 문제 |
|---|---|---|
| `## 제목` | h2 | OK |
| `**Q. …` | 굵은 문단(이 경우만 `**` 제거) | Q&A 전용 케이스만 |
| `- 항목` | li (단, `<ul>` 미감쌈) | 리스트 시맨틱 불완전 |
| `| 셀 | 셀 |` | **monospace 원문 출력** | 표가 `|---|`까지 글자로 노출 |
| 문단·리스트 내 `**굵게**` | 미변환 | `**굵게**` 글자 그대로 |
| `[링크](url)`·URL·`###`·인용 | 미지원 | 렌더 안 됨 |

- `prose-medirok` 클래스는 `globals.css`에 정의가 없어 실제 스타일도 없음.
- 표를 쓴 **기존 매거진 전체**(예: 강남 가격 가이드)도 동일 증상.

---

## 2. 해결 방향 — 옵션 A (채택)

**`react-markdown` + `remark-gfm`** 로 정식 마크다운 → React 엘리먼트 렌더.

근거:
- GFM 표·인라인 굵게·리스트·링크·인용·코드 모두 지원
- 기본적으로 **raw HTML 차단**(XSS 안전) — 별도 새니타이저 불필요
- `dangerouslySetInnerHTML` 미사용, Next SSR(서버 컴포넌트) 호환
- 정식 `<table>`·`<h2>`·`<strong>` 출력 → **AEO/GEO에도 유리**(답변엔진이 raw 파이프보다 정식 구조를 더 잘 인식)
- 변경 범위 작음: 재사용 컴포넌트 1개 + 페이지 파서 교체

(비교) 옵션 B(`marked`→HTML+sanitize)는 `dangerouslySetInnerHTML`+새니타이즈 관리 부담, 옵션 C(저장 시 변환)는 기존 데이터 마이그레이션 필요 → 모두 보류.

---

## 3. 구현 범위

### 3-1. 패키지 추가
```bash
npm i react-markdown remark-gfm
```

### 3-2. 재사용 컴포넌트 신설 — `src/components/Markdown.tsx`
- `react-markdown`에 `remarkPlugins={[remarkGfm]}` 적용
- `components` 매핑으로 각 요소를 **기존 Tailwind 디자인 토큰**에 연결(사이트 톤 유지):
  - `h2`/`h3` → 제목 스타일(현재 h2와 동일 톤)
  - `p` → 본문 문단
  - `strong` → 굵게
  - `ul`/`ol`/`li` → 리스트(불릿/들여쓰기)
  - **`table`/`thead`/`tbody`/`tr`/`th`/`td`** → 테두리·헤더 배경·셀 패딩(surface-border 토큰), 모바일 가로 스크롤 래퍼
  - `a` → 링크 색(accent), 외부 URL은 `target="_blank" rel="noopener nofollow"`
  - `blockquote`/`code` → 인용·코드 스타일
- raw HTML 비허용 기본값 유지(보안)

### 3-3. 매거진 페이지 교체 — `magazine/[slug]/page.tsx`
- 170~205행 수작업 파서 블록을 `<Markdown source={magazine.body} />` 한 줄로 대체
- `ShortAnswerBlock`·`FaqBlock`·`PriceTable`·`AuthorProfile` 등 나머지 구조는 그대로

### 3-4. (선택) 확장
- 같은 컴포넌트를 의원 `curationNote` 등 다른 마크다운 본문에도 재사용 가능

---

## 4. 고려사항

- **AEO/GEO**: 여전히 SSR로 HTML이 생성되므로 크롤·인용에 문제 없음. 오히려 정식 표/제목/강조로 구조가 명확해져 유리.
- **보안**: `react-markdown` 기본값은 raw HTML 미렌더 → 사용자/CMS 입력 안전.
- **성능**: 빌드 사이즈 소폭 증가(두 패키지). 본문 렌더는 서버에서 수행.
- **호환**: React 19 / Next 16에서 `react-markdown` 9.x + `remark-gfm` 4.x 동작.
- **회귀**: 기존 매거진(`**Q.`, 표 포함)들도 동일 컴포넌트로 정상 렌더 → 일괄 개선.

---

## 5. 작업 흐름 & 검증

1. `npm i react-markdown remark-gfm`
2. `src/components/Markdown.tsx` 작성(디자인 토큰 매핑)
3. `magazine/[slug]/page.tsx` 파서 → `<Markdown>` 교체
4. 이 글(`guide-geomdan-dental-2026`)로 표·굵게·링크 정상 렌더 확인
5. `tsc --noEmit` + `next build` 검증
6. 다른 매거진 1~2편도 렌더 확인(회귀 점검)
